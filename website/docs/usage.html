Usage & Workflows

Daemon control, polling, webhooks, manual triggers, retries, reports, and diagnostics.

#Commands at a glance

CommandDescription
claudear startStart the watcher daemon with all configured services
claudear stopStop the running watcher daemon
claudear statusShow the status of the running daemon
claudear pausePause the running watcher (stops processing new issues)
claudear resumeResume a paused watcher
claudear activityShow recent activity from the daemon
claudear seedMark all existing issues as seen (run this first!)
claudear dry-runShow what would be processed without running Claude
claudear pollStart polling all enabled sources (foreground, no daemon)
claudear webhookStart webhook server for real-time events
claudear triggerManually trigger a fix for an issue
claudear resetReset a failed attempt to allow retry
claudear statsShow statistics about fix attempts
claudear sourcesList configured sources
claudear dashboardStart the dashboard API server

claudear prsPull request management commands
claudear retriesRetry management commands
claudear inferenceInference analytics and management
claudear reportReport generation and scheduling
claudear diagDiagnostic commands for debugging

#Running Claudear

The recommended way to run Claudear is as a background daemon. It watches your configured sources, processes new issues automatically, and stays running until you tell it to stop.

# Start with polling + webhooks + dashboard
claudear start --poll --port 3100

# Custom poll interval
claudear start --poll --poll-interval 60000

# Worker-like mode (no webhooks/dashboard)
claudear start --poll --no-webhooks --no-dashboard

#Control commands

Once the daemon is running, you can check on it and control it without restarting:

claudear status
claudear pause
claudear resume
claudear activity
claudear activity 50
claudear stop

#Foreground polling

Runs in your terminal: useful for testing and debugging your setup.

claudear poll
claudear poll 60000
claudear poll --port 8080

#Webhooks

Webhooks let Claudear react to events instantly instead of waiting for the next poll. You can start a standalone webhook server, or use --setup-webhooks to automatically register webhooks with your providers and start listening.

claudear webhook
claudear webhook --setup-webhooks --base-url https://my-server.example.com:3100

#Triggering fixes manually

Sometimes you want to kick off a fix for a specific issue, or retry one that failed. You can do both from the CLI:

# Trigger a fix manually
claudear trigger linear abc123-def456
claudear trigger sentry 12345678

# Reset failed attempt for retry
claudear reset sentry 12345678

# Visibility helpers
claudear stats
claudear sources

#Managing PRs and retries

Track the pull requests Claudear has created, and manage the retry queue for issues that need another attempt.

# PR tracking
claudear prs list
claudear prs monitor
claudear prs monitor --continuous

# Retry queues
claudear retries list
claudear retries process

#Dashboard

Launch the web dashboard to monitor everything visually.

claudear dashboard
claudear dashboard 8080
claudear dashboard --dashboard-dir ./dashboard/dist

#Reports

Generate summary reports of what Claudear has been doing. Preview them locally, send them to your configured notification channels, or set up a recurring schedule.

claudear report preview daily
claudear report preview weekly
claudear report send daily
claudear report schedule --daily --hour 9
claudear report schedule --daily --weekly --hour 9

#Repository management

Claudear can discover your repos, track dependencies between them, and show what would be affected by changes.

claudear repos discover
claudear repos discover --paths ~/projects ~/work --save
claudear repos list
claudear repos index
claudear repos stats
claudear repos sync
claudear repos search "auth middleware"
claudear repos link my-lib my-app --dep-type npm
claudear repos graph
claudear repos graph --root my-lib
claudear repos cascade my-lib

#Analytics

See how Claudear is performing: which repos it targets, how confident its decisions are, and whether its choices are correct. You can also provide feedback to help improve future results.

claudear inference stats
claudear inference history
claudear inference history --limit 50
claudear inference feedback 42 --correct
claudear inference feedback 43 --actual-repo my-other-repo

#Troubleshooting

If something is not working as expected, these commands help you diagnose the problem. dry-run shows exactly what Claudear would process without actually running any fixes.

claudear diag db
claudear diag release-graph
claudear diag release-check owner/repo 42 --target owner/target-repo
claudear diag release-path owner/source-repo owner/target-repo
claudear dry-run

#Getting started sequence

  1. Run claudear seed to mark existing issues as seen (so Claudear doesn't re-process your backlog)
  2. Run claudear dry-run to preview what would be processed
  3. Start the daemon: claudear start --poll
  4. Enable webhooks for instant response times
  5. Turn on reports, regression monitoring, and advanced features as needed

#Pause and stop

  • claudear pause: stops picking up new work, keeps the process running so you can resume later
  • claudear stop: shuts down the daemon completely
