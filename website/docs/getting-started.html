Getting Started

Install Claudear, set up credentials, and run your first workflow.

#Install

#Homebrew (macOS/Linux)

brew tap abnegate/tap
brew install claudear

#APT (Debian/Ubuntu)

curl -fsSL https://abnegate.github.io/apt-repo/pubkey.gpg | sudo gpg --dearmor -o /usr/share/keyrings/claudear.gpg
echo "deb [signed-by=/usr/share/keyrings/claudear.gpg] https://abnegate.github.io/apt-repo stable main" | sudo tee /etc/apt/sources.list.d/claudear.list
sudo apt update && sudo apt install claudear

#Prebuilt binaries

Download binaries from the releases page for Linux and macOS (Intel/ARM) when you want a pinned build without local compilation.

#From source

git clone https://github.com/abnegate/claudear.git
cd claudear
cargo build --release
# Binary at target/release/claudear

#Docker image

docker pull ghcr.io/abnegate/claudear:latest

#What you need

Before you start, gather a few credentials:

  • An issue tracker token: a Linear API key, Sentry auth token, Jira API token, or similar, depending on where your issues live.
  • A GitHub or GitLab token: so Claudear can create branches and open pull requests on your behalf.
  • A notification channel: a Discord bot token, Slack bot token, or email credentials so you can receive updates.
  • Claude Code CLI: your AI agent runs through Claude Code. Make sure it's installed and authenticated.

#Quick start

Four commands to go from zero to your first automated fix. Seeding first is important: it tells Claudear to skip your existing issues so they don't all get processed at once.

# Copy the example config and fill in your tokens
cp claudear.example.toml claudear.toml

# Mark existing issues as already seen
claudear seed

# Start Claudear with polling and the dashboard
claudear start --poll --port 3100

# Open the dashboard
open http://localhost:3100

#Verify it works

Preview which issues Claudear would pick up without running anything:

claudear dry-run

Then create a tagged issue in your tracker and let it run. Confirm the full cycle:

  • The issue gets picked up
  • A pull request is created
  • You receive a notification
  • The dashboard shows the attempt

#Choose your runtime mode

#Background daemon (recommended)

claudear start runs in the background and is the best option for everyday use. Control it with status, pause, resume, and stop.

#Foreground polling

claudear poll runs in your terminal so you can see everything happening in real time. Useful for testing and debugging your configuration.

#Webhooks

claudear webhook listens for real-time events from your issue trackers instead of polling on an interval. Lower latency, but requires your server to be reachable from the internet.

#Tips for going live

  • Set up a reply channel. Connect Discord or Slack so the AI can ask you questions when it needs clarification.
  • Check the dashboard. Make sure it's accessible and showing data at your configured port.
  • Save your logs. Make sure logs are being written somewhere you can find them later.
  • Pause and resume as needed. claudear pause freezes processing without shutting down. claudear stop shuts everything down.
