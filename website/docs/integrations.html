Core

Integrations

Connect your issue trackers, source control, and notification channels to Claudear.

#Connecting your tools

Claudear connects to your existing tools in four categories: where issues come from, where code lives, where you get notified, and who you are across those platforms. You only need one of each to get started.

#Issue sources

These are the places Claudear picks up work from:

  • Linear: picks up issues matching your label, state, or assignee triggers. Webhooks for instant pickup.
  • Sentry: monitors top errors above configurable event count and escalation thresholds. Great for automatic bug triage.
  • Jira: Jira Cloud and Server/Data Center. Filter by project, status, label, issue type, or custom JQL.
  • GitHub Issues: native GitHub issue automation. Trigger on labels. Webhook support.
  • GitLab Issues: native GitLab issue and MR automation. Filter by label and state.
  • Discord: treat messages in a channel as issues. Useful for incident channels.
  • Slack: treat messages in a channel as issues. Useful for ad-hoc operational queues.

#Source control

Where Claudear creates pull requests:

  • GitHub (Personal Access Token): quickest to set up. Grant repo access and paste your token.
  • GitHub App: for organisations that prefer app-based auth with scoped installation access.
  • GitLab: personal access token + optional self-hosted base URL.

On GitHub and GitLab you can also configure a review_trigger tag (default: @claudear) — adding it to a PR review comment kicks off a reply from Claudear.

#Notifications

Where Claudear sends updates and questions.

#Reply-capable (recommended — lets you answer AI questions)

  • Discord: bot token + channel. Bi-directional.
  • Slack: bot token + channel. Bi-directional.
  • Email: SMTP for sending, IMAP for reading replies.

#Delivery-only

  • SMS (via Twilio)
  • Push (via Pushover)
  • WhatsApp
  • Telegram

Set up at least one reply-capable channel so Claudear can ask you questions when it gets stuck.

#Webhooks

Webhooks let Claudear react to events instantly rather than waiting for the next poll cycle. Use the CLI to automatically register webhooks with supported providers:

claudear webhook --setup-webhooks --base-url https://your-server.com

Always verify webhook secrets are set and check provider delivery logs after setup.

#User mapping

Map team members across all your platforms in [users.<slug>]. When an issue is assigned to someone, notifications route directly to them. Configure their identifiers once and Claudear handles the routing automatically.

#Recommended setup order

  1. One issue source + one source control provider + one notification channel
  2. Verify a complete end-to-end attempt works
  3. Add a reply-capable notification channel for AI questions
  4. Enable webhooks for instant pickup
  5. Add user mappings for targeted notifications
  6. Expand to additional sources and advanced features
